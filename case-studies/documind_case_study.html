Most "AI document tools" are just wrappers around a summarization prompt. DocuMind is a full RAG pipeline — extract, chunk, embed, retrieve, generate. Built from scratch.
The Problem
Documents are locked.

PDFs, invoices, contracts, research papers — critical information lives in unstructured files that can't be searched, queried, or actioned programmatically. Enterprise tools cost thousands. LLM wrappers hallucinate because they don't retrieve, they guess. The gap was a lightweight, accurate, composable pipeline that handles real-world documents and plugs into existing workflows.

Architecture
Nine steps. One pipeline.

Every document flows through the same deterministic path — no shortcuts, no black boxes.

01
Upload & Extract
PyMuPDF for native PDFs. Poppler + pytesseract OCR for scanned images and photo documents. Handles DOCX and TXT natively.
02
Document Classification
Groq LLM classifies each document into a semantic category — resume, contract, invoice, report, email, research paper — to tailor downstream extraction.
03
Semantic Chunking
Documents are split into overlapping semantic chunks rather than fixed token windows — preserving context across boundaries.
04
Vector Embeddings
each chunk is embedded using all-MiniLM-L6-v2 from sentence-transformers — a fast, high-quality 384-dim model optimised for semantic search.
05
ChromaDB Vector Store
Embeddings stored in ChromaDB with metadata — chunk index, document type, source page. Persists across sessions for re-querying without re-processing.
06
RAG Retrieval
User query is embedded at runtime. ChromaDB retrieves the top-k most semantically similar chunks — only relevant context reaches the LLM.
07
Groq LLM Generation
Retrieved chunks are injected into a structured prompt. Groq (LLaMA 3) generates grounded, evidence-backed answers — hallucination minimised by design.
08
Structured Output
Beyond Q&A: concise summary, JSON extraction of key fields, action items with owners, deadlines, and priorities.
09
Automation Layer
Action items auto-logged to Google Sheets. Summaries dispatched via Gmail. Flask webhook enables n8n / external trigger integration.
The Product
Ask anything. Get cited answers.

The "Chat with Document" tab lets users ask free-form questions about uploaded documents. Responses come with source citations — the exact chunks the LLM used to generate the answer. No black box.

Design Decisions
Why these choices matter.
01

Semantic chunking over fixed windows

Fixed token windows cut sentences mid-thought, destroying context. Semantic chunking with overlap preserves meaning at boundaries — retrieval quality is noticeably better on longer documents.

02

all-MiniLM-L6-v2 over larger models

A 384-dim model that runs fast locally with no API cost. For document retrieval at this scale, it matches much larger embedding models — choosing the right tool, not the most impressive-sounding one.

03

ChromaDB over in-memory vectors

ChromaDB persists embeddings to disk — re-querying the same document doesn't require re-processing. Critical for usability when documents take seconds to embed.

04

Classification before extraction

Knowing a document is a resume vs. a contract changes what you extract. Classification lets the prompt be document-type-aware — improving structured output quality significantly.

Learnings
What building this actually taught me.

RAG quality lives or dies by chunking strategy. Naive splitting produced retrieval misses that no amount of prompt engineering could fix — the fix was upstream.

OCR preprocessing is non-trivial. Deskewing and denoising before pytesseract significantly improved text quality on real-world scanned documents — garbage in, garbage out applies hard here.

Showing retrieved chunks to users builds trust. The source citation UI wasn't in the original spec — added it after noticing that users questioned AI answers without evidence. Transparency is UX.

The email output was the highest-leverage feature in demos. Stakeholders want summaries in their inbox, not dashboards they have to log into. Distribution matters as much as intelligence.

Modular architecture compounds. Because each layer is independent, adding the RAG pipeline to an existing extraction tool took two days, not two weeks. Design for replaceability.

DocuMind isn't a chatbot with a PDF attached. It's a full retrieval-augmented generation pipeline — where every answer is grounded in evidence, every insight triggers an action, and every component was chosen to solve a real problem.
