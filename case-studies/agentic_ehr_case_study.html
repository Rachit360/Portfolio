Node.js React MongoDB Ollama llama3.2-vision JWT Auth Express Ghostscript
Medical records are complex, unstructured, and critical. I built a system where AI doesn't just read them — it reasons over them, flags risks, recommends next steps, and remembers every patient.
The Problem
Doctors drown in paperwork.

A doctor seeing 30 patients a day can't deeply read every uploaded report. Scanned PDFs are especially painful — they're images, not text. Critical findings get buried. Follow-up recommendations exist in prose that no system can act on. The gap is a system that reads, understands, and structures medical documents — automatically — while building a persistent memory of each patient over time.

Beyond that: EHR systems are notoriously rigid. Most are built for one clinic workflow. Agentic-EHR was designed to show that a small, well-architected full-stack system with an agentic core can do what expensive enterprise software can't — reason, not just store.

The AI Agent Pipeline
Five steps. Sequential. Stateful.

Every uploaded report triggers a deterministic 5-step agentic pipeline. Each step's output feeds the next — no step is skipped, no context is lost.

01
Extract
llama3.2-vision reads the scanned PDF page by page via Ghostscript-rendered images. Unlike text-based OCR, the vision model understands layout — tables, labels, handwritten annotations. Raw medical text is extracted with structural context preserved.
llama3.2-vision · Ghostscript PDF→image · Ollama local inference
02
Summarize
The extracted text is passed back through the LLM with a structured clinical prompt. Output is a 5-section summary: Chief Complaint, Findings, Diagnosis, Medications, and Patient History. Consistent format, regardless of the original document structure.
Structured prompt · 5-section clinical output · JSON-serializable
03
Flag
A dedicated flagging pass identifies clinically significant signals — abnormal vitals, drug interactions, critical lab values, urgent diagnoses. Output is structured JSON with severity levels: Critical / Warning / Info. These drive the real-time alert UI in the dashboard.
Severity-tagged JSON · Critical / Warning / Info · Real-time alert UI
04
Next Steps
Based on the summary and flags, the LLM generates actionable clinical recommendations — follow-up tests, specialist referrals, medication adjustments, lifestyle advice. Not generic suggestions; grounded in the specific findings of this patient's report.
Context-aware recommendations · Grounded in per-report findings
05
Memory
The final step updates a persistent AI profile for each patient in MongoDB. Every report processed adds to a longitudinal record — symptoms, diagnoses, medications, flags — so the AI chat has full patient context across all visits, not just the current document.
MongoDB stateful memory · Longitudinal patient profile · Context-aware chat
"Most AI demos process one document and forget it. Agentic-EHR's memory step means every interaction makes the system smarter about that patient — it compounds over time."
Full-Stack Architecture
Not a script. A product.

Agentic-EHR is a complete full-stack application — not a Python notebook or a demo widget. It has real auth, real data persistence, real user flows.

🔐
Auth Layer
JWT-based doctor login and signup. Protected routes via Express middleware. Tokens validated on every request — no anonymous access to patient data.
🧠
AI Agent Service
aiAgentService.js orchestrates all 5 pipeline steps. Runs sequentially, each step receiving the output of the previous. The brain of the system.
🗄️
MongoDB Models
Three collections: User (doctors), Patient (profiles + AI memory), Report (raw + processed). Stateful AI profiles stored as structured documents.
⚛️
React Frontend
Dashboard, Patients, PatientDetail, Reports pages. AuthContext for global state. All API calls through a centralized api.js service layer.
💬
AI Chat
Chat interface uses the patient's full MongoDB memory as context. The LLM answers questions about the patient's entire history — not just the last uploaded report.
🚨
Critical Alerts
Flags from Step 3 surface as real-time severity alerts in the dashboard. Critical findings are impossible to miss — the UI is designed around clinical urgency.
The Product
A dashboard built for clinical urgency.

The frontend is designed around the doctor's mental model — patients front and center, critical flags surfaced immediately, AI recommendations one click away.

Design Decision
Why local inference matters here.

Medical data is sensitive. Running llama3.2-vision via Ollama means no patient data leaves the machine — no API calls to OpenAI, no third-party logging, no data retention concerns. This was a deliberate architectural choice, not a cost-saving workaround.

It also makes the system deployable in restricted healthcare environments — hospitals and clinics that can't pipe patient data to external LLM providers can still benefit from AI-assisted analysis. Local inference is the only viable path in regulated industries.

Learnings
What this build actually taught me.

Sequential agentic pipelines are more reliable than parallel ones when each step's output is semantically meaningful to the next. The summary makes the flagging better. The flags make the recommendations more specific.

Vision models change what's possible with documents. llama3.2-vision didn't just extract text — it understood layout. Tables, headers, and annotated fields came through structurally intact, which no traditional OCR pipeline could match.

Stateful memory is what separates a demo from a product. A system that forgets patients after one interaction isn't useful in healthcare. MongoDB as a long-term AI memory store was the design decision that made this feel real.

Prompt engineering is domain-specific. Generic summarization prompts produced generic outputs. Clinical prompts structured around Chief Complaint / Findings / Diagnosis produced outputs a doctor could actually act on.

Building full-stack meant owning the whole user experience. Auth flows, loading states, alert hierarchy, chat UX — every layer mattered. AI pipeline quality only matters if the product is usable.

Agentic-EHR isn't a proof of concept — it's a working product that demonstrates agentic AI in a high-stakes domain, with real auth, real persistence, and a pipeline that reasons rather than retrieves.
